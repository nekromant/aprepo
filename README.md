# aprepo

**The problem**: ever since the world entered the era of political turmoil, android infrastructure split in multiple stores. Now you have to keep 2 to 3 app stores (at least play store + f-droid for opensource, sometimes appgallery or rustore, depending on where you are). Since only one (play store) really supports installing updates in background, you're stuck manually clicking to install updates and waste time. Since developer of some stores are retarded enough to add ads to the already pretty frustrating process, something was needed to be done. 

**The solution:** Arm your own server with **aprepo**. Aprepo will download the apps you use and their updates automatically from different stores and post it as an f-droid repo. Finally you can have just one appstore for everything and let f-droid privileged extension install everything in backround. As a side effect - you can use this stuff to shield your kids from ever experiencing the whole ADHD-stimulating environment of mobile shit.

**How it works:** Aprepo downloads all the required APKs/XAPKS from stores and places in folder. xapk are repacked into apks as needed (a workaround, since f-droid doesn't yet support installing split APKs). Then you use fdroidserver to publish a repo. 

## Features

- **Multiple download sources**: Google Play, RuStore, APKPure, GitHub releases, direct HTTP/WebDL
- **Per-architecture downloads**: Supports architecture-specific downloads with automatic fallback to universal
- **Smart throttling**: Two policies — `metadata` (version-aware, skips unchanged downloads) and `dumb` (time-based re-download)
- **XAPK repacking**: Extracts and merges XAPK bundles (base APK + architecture splits) into standalone APKs
- **Retention management**: Automatically purges old versions exceeding configured depth
- **State persistence**: Tracks downloaded versions and source capabilities in `state.yaml`

## Building

Requirements:
- Rust 1.78+ (edition 2021)

```bash
git clone <repo-url> aprepo
cd aprepo
cargo build --release
```

The binary will be at `target/release/aprepo`.

## Running

### Quick start

```bash
# Bootstrap a config.yaml and a p12 signature file
./target/release/aprepo --config config.yaml bootstrap

#Now, edit config.yaml to include the actual packages 

# Default: download then process
./target/release/aprepo --config config.yaml

# Download only
./target/release/aprepo --config config.yaml download

# Process cached files only
./target/release/aprepo --config config.yaml process

# Force re-download / re-process
./target/release/aprepo --config config.yaml --force download

# Filter to a single package
./target/release/aprepo --config config.yaml --package com.example.app download
```

### Example configuration

```yaml
settings:
  cache_dir: "./cache"
  output_dir: "./output"
  retention_depth: 2
  architectures:
    - arm64-v8a
    - armeabi-v7a
  repack_xapk: true

sources:
  google_play:
    throttle_policy: dumb
    throttle_interval: 1d
    delay_between_requests: 2s
    token: "$PLAYSTORE_TOKEN"
    packages:
      - com.example.app

  github:
    throttle_policy: metadata
    throttle_interval: 1h
    delay_between_requests: 1s
    token: "$GITHUB_TOKEN"
    packages:
      - repo: "owner/repo"
        mask: "*.apk"
        arch_masks:
          arm64-v8a: "*arm64*"
          armeabi-v7a: "*armeabi*"

  webdl:
    throttle_policy: metadata
    throttle_interval: 1h
    delay_between_requests: 1s
    packages:
      - filename: "app.apk"
        url: "https://example.com/app.apk"
        arch_files:
          arm64-v8a:
            filename: "app-arm64.apk"
            url: "https://example.com/app-arm64.apk"
```

### Environment variables

The configuration supports `$VARIABLE` interpolation. Required variables must be set in your environment:

```bash
export PLAYSTORE_TOKEN="your_token"
export GITHUB_TOKEN="your_token"
./target/release/aprepo --config config.yaml download
```

## Backend dependencies

- **Google Play / RuStore / APKPure**: Requires `apkeep` installed. For RuStore support, use the fork from [https://github.com/Foxushka/apkeep](https://github.com/Foxushka/apkeep) — this backend is not yet included in the upstream master branch.
- **GitHub**: Requires the `gh` CLI tool installed and authenticated.
- **XAPK repack** (optional): `zipalign` and `apksigner` from the Android SDK for alignment and signing.
